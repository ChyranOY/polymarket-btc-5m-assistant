<feature>
  <name>Fee-Aware Position Sizing</name>
  <files>src/domain/sizing.js, test/domain/sizing.test.js</files>
  <behavior>
    computeTradeSizeWithFees(balance, config, feeRateBps) extends existing sizing with fee deduction:
      balance=1000, stakePct=0.08, feeRateBps=200 (2%) -> sizeUsd = 80 * (1 - 200/10000) = 80 * 0.98 = 78.40
      balance=1000, stakePct=0.08, feeRateBps=null -> falls back to computeTradeSize (no fee deduction)
      balance=1000, stakePct=0.08, feeRateBps=0 -> sizeUsd = 80 (no deduction)
      balance=50, stakePct=0.08, feeRateBps=200 -> sizeUsd = 3.92 (below minTradeUsd=25) -> returns 0
      feeRateBps > 1000 (10%) -> clamp to 1000 to prevent absurd deductions (safety valve)

    Existing computeTradeSize remains unchanged for backward compatibility.
  </behavior>
  <implementation>
    Add computeTradeSizeWithFees to sizing.js. It calls computeTradeSize first, then applies fee deduction.
    Fee deduction: effectiveSize = rawSize * (1 - Math.min(feeRateBps, 1000) / 10000).
    Re-apply min/max clamping after fee deduction.
    Pure function, no I/O.
  </implementation>
</feature>

<feature>
  <name>Kill-Switch Domain Logic</name>
  <files>src/domain/killSwitch.js, test/domain/killSwitch.test.js</files>
  <behavior>
    createKillSwitchState() returns initial state:
      -> { active: false, overrideActive: false, overrideCount: 0, overrideLog: [], lastResetDate: null }

    checkKillSwitch(state, todayRealizedPnl, maxDailyLossUsd) checks activation:
      todayRealizedPnl=-50, maxDailyLossUsd=50 -> { triggered: true, reason: 'Daily loss -$50.00 hit limit -$50.00' }
      todayRealizedPnl=-49, maxDailyLossUsd=50 -> { triggered: false }
      todayRealizedPnl=-50, maxDailyLossUsd=50, state.overrideActive=true -> { triggered: false, overridden: true }
      todayRealizedPnl=-60, maxDailyLossUsd=50, state.overrideActive=true -> { triggered: true, reason: 'Daily loss -$60.00 exceeded override threshold -$55.00' }
        (override allows 10% more loss buffer, then re-triggers)
      maxDailyLossUsd=0 or null -> { triggered: false } (kill-switch disabled)

    overrideKillSwitch(state) activates override:
      -> new state with overrideActive=true, overrideCount++, overrideLog appended with timestamp
      Does NOT clear todayRealizedPnl (losses persist, just allows more trading)

    shouldResetKillSwitch(state, nowDate, timezone='America/Los_Angeles') checks midnight reset:
      lastResetDate='2026-02-22', nowDate is 2026-02-23 in PT -> true
      lastResetDate='2026-02-23', nowDate is 2026-02-23 in PT -> false
      lastResetDate=null -> true (first run)

    resetKillSwitch(state) resets for new day:
      -> new state with overrideActive=false, overrideCount=0, lastResetDate=today
      (overrideLog preserved for audit trail)
  </behavior>
  <implementation>
    Pure functions operating on plain state objects. No I/O, no side effects.
    Override allows additional 10% loss buffer beyond the limit (configurable).
    Midnight PT detection uses Intl.DateTimeFormat with America/Los_Angeles timezone.
    Override re-trigger: if losses continue after override, the new effective limit = maxDailyLossUsd * 1.1.
    If losses exceed that, kill-switch re-triggers even with override active.
  </implementation>
</feature>
